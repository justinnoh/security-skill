# API 오용 (A-01 ~ A-02)

## A-01 DNS lookup에 의존한 보안 결정 (CWE-350)
- **목적**: DNS 이름(역방향 조회 등)을 신뢰해 보안 결정을 내리지 않게.
- **공격 시나리오**: 역방향 DNS가 `*.trusted.com`이면 허용 → 공격자가 PTR 조작/스푸핑으로 우회.
- **탐지 위치**: 접근제어/인가에 호스트명·역방향 DNS 결과 사용, `gethostbyaddr` 기반 허용.
- **체크 방법**: 보안 결정이 DNS 조회 결과에 의존하는지.
- **PASS**: IP 화이트리스트·상호 TLS·서명 토큰 등 위조 곤란한 식별자 사용.
- **VULNERABLE**: 호스트명/역방향 DNS로 신뢰 판단.
- **수정**: 암호학적 인증(mTLS/토큰), 고정 IP/네트워크 정책.

## A-02 취약한 API 사용 (CWE-477)
- **목적**: 폐기/위험/비권장 API 사용을 제거.
- **공격 시나리오**: deprecated·취약 함수(`gets`, 약한 암호 API, 취약 버전 라이브러리) 사용.
- **탐지 위치**: deprecation 경고, 알려진 위험 함수, `npm audit`/SCA 결과, 구버전 의존성.
- **체크 방법**: 폐기/취약 API 및 알려진 취약점 있는 의존성 사용 여부.
- **PASS**: 권장 API 사용, 의존성 취약점 없음(audit 통과).
- **VULNERABLE**: 폐기/취약 API 또는 알려진 취약 의존성 사용.
- **NEEDS_REVIEW**: 취약점은 있으나 영향 경로 미확정인 전이 의존성.
- **수정**: 권장 대체 API, 의존성 업그레이드, SCA를 CI에 통합.
