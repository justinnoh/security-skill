# 시간 및 상태 (T-01 ~ T-02)

## T-01 경쟁조건: 검사시점과 사용시점 / TOCTOU (CWE-367)
- **목적**: 검사(check)와 사용(use) 사이 상태 변경으로 인한 우회/오류 방지.
- **공격 시나리오**: 잔액 확인 후 차감 사이 동시 요청으로 이중 인출; 파일 존재 확인 후 심볼릭 링크 교체.
- **탐지 위치**: 잔액/재고/쿠폰 차감, 파일 `exists→open`, 비원자적 read-modify-write.
- **체크 방법**: 동시성 자원 갱신이 트랜잭션/락/원자연산 없이 일어나는지.
- **PASS**: DB 트랜잭션+행 잠금 또는 원자적 갱신(`UPDATE ... WHERE balance>=?`), 멱등성 키.
- **VULNERABLE**: 검사와 갱신이 분리되고 락/트랜잭션 없음.
- **NEEDS_REVIEW**: 단일 인스턴스 가정의 인메모리 락(수평 확장 시 깨짐).
- **수정**: 원자적 SQL, `SELECT ... FOR UPDATE`, 분산 락, 멱등 처리.

## T-02 종료되지 않는 반복문 또는 재귀 함수 (CWE-835)
- **목적**: 무한 루프/무한 재귀로 인한 DoS 방지.
- **공격 시나리오**: 외부 입력으로 종료 조건이 충족되지 않는 루프/재귀 → CPU 고갈.
- **탐지 위치**: `while(true)`, 입력 기반 재귀 깊이, 페이지네이션 루프, 정규식(ReDoS).
- **체크 방법**: 반복/재귀의 종료 조건이 외부 입력에 안전하게 의존하는지, 상한이 있는지.
- **PASS**: 명확한 종료 조건+반복 상한/타임아웃, 재귀 깊이 제한, 안전 정규식.
- **VULNERABLE**: 종료 보장 없는 루프/재귀, 입력 길이 비례 폭증 정규식.
- **수정**: 반복 상한·타임아웃, 반복 변환, ReDoS 안전 패턴/라이브러리.
