# 보안기능 (S-01 ~ S-16)

인증·인가·암호·비밀번호·세션 관련 보안약점. 각 항목은
**목적 / 공격 시나리오 / 탐지 위치 / 체크 방법 / PASS / VULNERABLE / NEEDS_REVIEW / 수정 / 예시**로 진단한다.

---

## S-01 적절한 인증 없는 중요기능 허용 (CWE-306)
- **목적**: 중요기능/자원 접근 전 인증 강제.
- **공격 시나리오**: 인증 미들웨어 누락 라우트로 비인증 접근.
- **탐지 위치**: 라우트 정의, 인증 미들웨어 적용 여부.
- **PASS**: 모든 중요 라우트에 인증 미들웨어 적용, 우회 경로 없음.
- **VULNERABLE**: 중요기능이 비인증 접근 가능(예: 공개 업로드/관리 엔드포인트).
- **수정**: 라우트별 인증 강제, 기본 차단(deny-by-default).

## S-02 부적절한 인가 / IDOR·BOLA (CWE-285)
- **목적**: 인증된 사용자가 권한 밖 자원에 접근하지 못하게.
- **공격 시나리오**: `/orders/123`의 id를 바꿔 타인 데이터 조회.
- **탐지 위치**: id 파라미터로 자원 조회하는 모든 핸들러.
- **체크 방법**: 조회/수정 전 소유권·조직 멤버십 검증 존재 여부.
- **PASS**: 모든 객체 접근에 소유권/role 검증.
- **VULNERABLE**: id만으로 조회하고 소유권 미검증.
- **수정**: `WHERE id=? AND owner_id=?`, 정책 기반 인가 계층.

## S-03 중요한 자원에 대한 잘못된 권한 설정 (CWE-732)
- **목적**: 파일/디렉토리/객체 스토리지/DB 권한 최소화.
- **공격 시나리오**: 파일 0777, S3 버킷 public-read, 과도한 DB 권한.
- **탐지 위치**: 파일 생성 권한, 클라우드 스토리지 ACL, DB 계정 권한.
- **PASS**: 최소권한, 비공개 기본값.
- **VULNERABLE**: world-writable/public ACL, 광범위 DB 권한.
- **수정**: 최소권한 부여, private 기본, 서명 URL.

## S-04 취약한 암호화 알고리즘 사용 (CWE-327)
- **공격 시나리오**: MD5/SHA1/DES/RC4/ECB로 약화된 암호.
- **탐지 위치**: `crypto.createHash('md5')`, `createCipheriv('des'..)`, `aes-...-ecb`.
- **PASS**: AES-GCM 등 표준 + 안전 모드, 해시는 SHA-256↑.
- **VULNERABLE**: MD5/SHA1(보안용)/DES/RC4/ECB 사용.
- **수정**: AES-256-GCM, SHA-256/512, 검증된 라이브러리.

## S-05 암호화되지 않은 중요정보 (평문 저장/전송) (CWE-311/312/319)
- **공격 시나리오**: 비밀번호/주민번호/카드 평문 저장, HTTP 전송.
- **탐지 위치**: DB 모델 민감 컬럼, 로그, HTTP 클라이언트, TLS 설정.
- **PASS**: 저장 시 암호화/해시, 전송 시 TLS 강제.
- **VULNERABLE**: 민감정보 평문 저장 또는 평문 전송.
- **수정**: 저장 암호화, 전 구간 HTTPS, 로그 마스킹.

## S-06 하드코드된 중요정보 (CWE-798)
- **공격 시나리오**: 소스/리포에 API 키·비밀번호·토큰 하드코딩.
- **탐지 위치**: 소스 전체 grep(`password=`, `secret`, `apikey`, `BEGIN PRIVATE KEY`), 기본값 fallback.
- **PASS**: 모든 비밀값을 환경변수/시크릿 매니저로 주입, fallback 비밀값 없음.
- **VULNERABLE**: 코드에 비밀값 또는 `process.env.X || '하드코드'`.
- **수정**: 시크릿 매니저, env 강제(미설정 시 기동 실패), 이력 정리.

## S-07 충분하지 않은 키 길이 사용 (CWE-326)
- **탐지 위치**: 키 생성 파라미터(RSA modulus, AES 키 길이).
- **PASS**: RSA≥2048, AES≥128(권장 256), ECC≥256.
- **VULNERABLE**: RSA<2048, DES(56), AES 약식 키.
- **수정**: 권장 키 길이로 상향.

## S-08 적절하지 않은 난수 값 사용 (CWE-330)
- **공격 시나리오**: 예측 가능 토큰/세션ID/비밀번호 재설정 토큰.
- **탐지 위치**: `Math.random()`로 보안 토큰 생성.
- **PASS**: CSPRNG(`crypto.randomBytes`, `randomUUID`) 사용.
- **VULNERABLE**: `Math.random()` 등 비암호학 난수로 보안값 생성.
- **수정**: `crypto.randomBytes`/`randomUUID`.

## S-09 취약한 비밀번호 허용 (CWE-521)
- **탐지 위치**: 회원가입/변경 시 비밀번호 정책 검증.
- **PASS**: 최소 길이/복잡도/유출목록 검증.
- **VULNERABLE**: 정책 없음(짧은/단순 비밀번호 허용).
- **수정**: 길이·복잡도 정책, HIBP 등 유출 비밀번호 차단.

## S-10 부적절한 전자서명 확인 (CWE-347)
- **공격 시나리오**: JWT `alg:none`, 웹훅 서명 미검증, 서명 검증 생략.
- **탐지 위치**: JWT 검증 옵션, 웹훅 핸들러(Stripe/GitHub 등).
- **PASS**: 서명·알고리즘 화이트리스트 검증, 웹훅 HMAC 검증.
- **VULNERABLE**: 서명 미검증 또는 `alg` 미고정.
- **수정**: 알고리즘 고정, 서명 필수 검증, 타이밍 안전 비교.

## S-11 부적절한 인증서 유효성 검증 (CWE-295)
- **공격 시나리오**: TLS 인증서 검증 비활성으로 MITM.
- **탐지 위치**: `rejectUnauthorized:false`, `NODE_TLS_REJECT_UNAUTHORIZED=0`, `verify=False`.
- **PASS**: 인증서 검증 활성, 신뢰 체인 확인.
- **VULNERABLE**: 인증서 검증 비활성화.
- **수정**: 검증 활성, 사설 CA는 신뢰스토어 등록.

## S-12 사용자 하드디스크에 저장되는 쿠키를 통한 정보 노출 (CWE-525)
- **공격 시나리오**: 민감정보를 영속 쿠키/localStorage에 평문 저장.
- **탐지 위치**: `Set-Cookie`(만료 길고 민감값), `localStorage.setItem(token..)`.
- **PASS**: 세션쿠키 + HttpOnly/Secure/SameSite, 민감정보 클라이언트 저장 안 함.
- **VULNERABLE**: 민감정보를 영속 쿠키/스토리지에 저장.
- **수정**: 서버 세션, HttpOnly 쿠키, 클라이언트 저장 최소화.

## S-13 주석문 안에 포함된 시스템 주요정보 (CWE-615)
- **탐지 위치**: 주석 grep(`// password`, `TODO 계정`, 내부 URL/IP/키).
- **PASS**: 주석에 비밀/내부정보 없음.
- **VULNERABLE**: 주석에 계정·키·내부 엔드포인트 노출.
- **수정**: 주석 제거, 빌드시 주석 스트립.

## S-14 솔트 없이 일방향 해시 함수 사용 (CWE-759/760)
- **공격 시나리오**: 솔트 없는 해시 → 레인보우 테이블 공격.
- **탐지 위치**: 비밀번호 저장 해시 로직.
- **PASS**: bcrypt/scrypt/argon2(솔트 내장) 또는 사용자별 솔트+stretching.
- **VULNERABLE**: `sha256(password)` 등 무솔트/고속 해시.
- **수정**: bcrypt/argon2id 적용.

## S-15 무결성 검사 없는 코드 다운로드 (CWE-494)
- **공격 시나리오**: 런타임에 원격 스크립트/플러그인을 무결성 검증 없이 받아 실행.
- **탐지 위치**: 동적 스크립트 로드, 외부 바이너리/업데이트 다운로드.
- **PASS**: 서명/해시(SRI) 검증 후 실행, HTTPS.
- **VULNERABLE**: 무결성 검증 없이 원격 코드 실행.
- **수정**: 체크섬/서명 검증, SRI, 고정 버전.

## S-16 반복된 인증시도 제한 기능 부재 (CWE-307)
- **공격 시나리오**: 무차별 대입으로 비밀번호/OTP 추측.
- **탐지 위치**: 로그인/OTP/비밀번호 재설정 엔드포인트(범용 API rate limit과 별개).
- **PASS**: 시도 횟수 제한·계정 잠금·지연·CAPTCHA.
- **VULNERABLE**: 인증 시도 제한 없음.
- **수정**: 시도 카운트·잠금·백오프, IP/계정 단위 제한.
