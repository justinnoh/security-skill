# 코드오류 (C-01 ~ C-05)

## C-01 Null Pointer 역참조 (CWE-476)
- **목적**: null/undefined 역참조로 인한 비정상 종료(DoS) 방지.
- **공격 시나리오**: 입력 누락 시 `obj.prop` 접근으로 크래시.
- **탐지 위치**: 외부 입력/조회 결과를 null 검사 없이 역참조.
- **PASS**: null 검사·옵셔널 체이닝·기본값, 조회 결과 존재 확인.
- **VULNERABLE**: null 가능 값을 검사 없이 역참조.
- **수정**: 가드/`?.`/기본값, 스키마 검증.

## C-02 부적절한 자원 해제 (CWE-404/772)
- **목적**: 파일/소켓/DB 커넥션/핸들 누수 방지.
- **공격 시나리오**: 예외 경로에서 커넥션 미반환 → 풀 고갈.
- **탐지 위치**: DB 커넥션, 파일 핸들, 스트림의 close/release 경로.
- **PASS**: `finally`/`using`/풀 자동 반환으로 항상 해제.
- **VULNERABLE**: 오류 경로에서 자원 미해제.
- **수정**: try/finally, 커넥션 풀, 자동 정리 패턴.

## C-03 해제된 자원 사용 (CWE-416, Use-After-Free)
- **목적**: 이미 해제/종료된 자원 재사용 방지.
- **공격 시나리오**: close된 커넥션/스트림 재사용, 네이티브 UAF.
- **탐지 위치**: close 후 동일 핸들 사용, 해제 후 콜백 접근.
- **PASS**: 해제 후 참조 무효화, 상태 플래그 확인.
- **VULNERABLE**: 해제된 자원 재사용.
- **수정**: 참조 null 처리, 생명주기 관리.

## C-04 초기화되지 않은 변수 사용 (CWE-457)
- **목적**: 미초기화 값 사용으로 인한 예측불가/정보노출 방지.
- **탐지 위치**: 조건 분기에서만 초기화되는 변수, 미초기화 버퍼.
- **PASS**: 선언 시 초기화, 모든 경로 초기화 보장, `Buffer.alloc`.
- **VULNERABLE**: 일부 경로에서 미초기화 사용, `allocUnsafe` 노출.
- **수정**: 기본값 초기화, 엄격 모드/린터, 제로화 할당.

## C-05 신뢰할 수 없는 데이터의 역직렬화 (CWE-502)
- **목적**: 신뢰 불가 입력 역직렬화로 인한 RCE/객체주입 방지.
- **공격 시나리오**: `pickle`/`unserialize`/Java 역직렬화, 프로토타입 오염.
- **탐지 위치**: `JSON.parse`+`Object.assign` 병합, `node-serialize`, YAML `load`, `__proto__`.
- **PASS**: 신뢰 데이터만 역직렬화, 스키마 검증, 안전 파서, 프로토타입 오염 방어.
- **VULNERABLE**: 신뢰 불가 입력을 위험 역직렬화 API로 처리.
- **수정**: 안전 포맷(JSON+스키마), 허용 타입 제한, `Object.create(null)`, 검증 라이브러리.
