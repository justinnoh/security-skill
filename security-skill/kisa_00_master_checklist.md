# KISA 2021 마스터 체크리스트 (Master Checklist)

이 파일은 「소프트웨어 보안약점 진단가이드(2021, KISA/행정안전부)」의
**설계단계 보안설계 기준 20개 항목**과 **구현단계 보안약점 49개 항목**을
번호로 고정한 단일 출처(single source of truth)입니다.

진단 시 모든 항목은 반드시 `PASS` / `VULNERABLE` / `NEEDS_REVIEW` 중 하나로 판정합니다.
항목별 상세 필드(목적·공격 시나리오·탐지 위치·체크 방법·판정 기준·수정 가이드·예시 코드)는
각 카테고리 파일에 정의되어 있으며, 이 표의 "상세 파일" 열을 따릅니다.

## 판정 규칙

- **PASS**: 해당 약점에 대한 방어가 코드/설계에 명확히 존재하고 우회 경로가 없음.
- **VULNERABLE**: 약점이 실제로 존재하거나 방어가 없음. 즉시 수정 대상.
- **NEEDS_REVIEW**: 코드만으로 판단 불가(런타임 설정 의존, 외부 라이브러리 내부 동작 등) → 수동 확인 필요.
- **N/A**: 해당 기술스택에 존재할 수 없는 약점(예: 메모리 안전 언어의 버퍼 오버플로우). 사유를 반드시 명시.

전체 진단 결과는 `12_report_template.md`의 49개 항목 결과표로 강제 출력한다.
