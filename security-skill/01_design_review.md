# 설계단계 보안설계 진단 (D-01 ~ D-20)

KISA 2021 설계단계 기준. **설계 산출물**(요구사항 정의서, 아키텍처/상세설계서, API 명세,
DB 스키마, 시퀀스 다이어그램)에서 각 항목이 반영되었는지 확인한다.
코드만 있고 설계 문서가 없으면, 코드 구조에서 해당 통제의 존재 여부로 대체 판정하고
`NEEDS_REVIEW`로 표시한다.

각 항목 공통 판정:
- **PASS**: 설계에 통제가 명시되고 일관되게 적용됨
- **VULNERABLE**: 통제가 누락되었거나 잘못 설계됨
- **NEEDS_REVIEW**: 설계 산출물 부재 또는 구현 의존으로 확인 필요

---

## 입력데이터 검증 및 표현

### D-01 DBMS 조회 및 결과 검증
- **목적**: DB 질의에 외부 입력이 안전하게 반영되고, 결과가 신뢰 후 사용되는지.
- **체크**: 모든 DB 접근이 파라미터 바인딩/ORM을 사용하도록 설계되었는가. 동적 쿼리 조립 지점이 명세되어 있는가.
- **VULNERABLE 기준**: 문자열 결합 쿼리를 허용하는 설계, 입력 검증 책임 주체 미지정.

### D-02 XML 조회 및 결과 검증
- **목적**: XPath/XQuery/XML 파싱 시 외부 입력 검증.
- **체크**: XML 파서의 외부 개체(DTD/Entity) 비활성화가 설계 표준에 포함됐는가.

### D-03 디렉토리 서비스 조회 및 결과 검증
- **목적**: LDAP 등 디렉토리 질의에 입력값이 안전하게 반영.
- **체크**: DN/필터 특수문자 이스케이프 규칙이 설계에 정의됐는가.

### D-04 시스템 자원 접근 및 명령어 수행 입력값 검증
- **목적**: 파일 경로·OS 명령 실행 입력 검증.
- **체크**: 화이트리스트 기반 경로/명령 정책, 셸 호출 금지 원칙이 설계됐는가.

### D-05 웹 서비스 요청 및 결과 검증
- **목적**: 외부 API/웹서비스 호출(SSRF 포함) 대상 제한.
- **체크**: 외부 요청 대상 허용목록, 내부망 접근 차단이 설계됐는가.

### D-06 웹 기반 중요기능 수행 요청 유효성 검증 (CSRF 방지)
- **목적**: 상태변경 요청의 출처 검증.
- **체크**: CSRF 토큰/SameSite 쿠키/재인증 설계 여부.

### D-07 HTTP 프로토콜 유효성 검증
- **목적**: 헤더/쿠키/리다이렉트에 CRLF 등 미검증 입력 반영 방지.
- **체크**: 응답 헤더에 입력값 직접 반영 금지 규칙이 있는가(HTTP 응답분할 방지).

### D-08 허용된 범위 내 메모리 접근
- **목적**: 버퍼/배열 경계 검증(C/C++ 등).
- **체크**: 경계 검사·안전 함수 사용 표준. (메모리 안전 언어면 N/A 가능)

### D-09 보안기능 동작에 사용되는 입력값 검증
- **목적**: 인증/인가/권한 판단에 쓰이는 입력의 신뢰성.
- **체크**: 보안 결정에 클라이언트 제공값(role, isAdmin 등)을 그대로 신뢰하지 않도록 설계.

### D-10 업로드·다운로드 파일 검증
- **목적**: 파일 확장자/타입/크기/저장위치 통제.
- **체크**: 화이트리스트 확장자, 실행권한 없는 저장소, 다운로드 경로 검증 설계.

## 보안기능

### D-11 인증 대상 및 방식
- **체크**: 모든 중요기능/자원 접근 전 인증 요구가 설계됐는가. 인증 우회 경로 부재.

### D-12 인증 수행 제한 (반복 인증시도 제한)
- **체크**: 로그인/OTP 등 인증 시도 횟수 제한·계정 잠금·지연 설계.

### D-13 비밀번호 관리
- **체크**: 복잡도/길이 정책, 솔트 적용 해시 저장, 변경/재설정 절차 설계.

### D-14 중요자원 접근통제
- **체크**: 자원별 권한 모델(소유권/조직 멤버십)과 서버측 인가 설계.

### D-15 암호키 관리
- **체크**: 키 생성/보관/교체/폐기 절차, 키를 코드/저장소에 두지 않는 설계.

### D-16 암호연산
- **체크**: 검증된 표준 알고리즘/모드 사용, 자체 암호 구현 금지 설계.

### D-17 중요정보 저장
- **체크**: 개인정보/인증정보 저장 시 암호화 및 접근통제 설계.

### D-18 중요정보 전송
- **체크**: 전 구간 TLS, 민감정보 URL/로그 노출 금지 설계.

## 에러처리

### D-19 예외처리
- **체크**: 일관된 예외 처리 정책, 내부정보 비노출 오류 응답 설계.

## 세션통제

### D-20 세션 통제
- **체크**: 세션 식별자 재발급(로그인 시), 만료/무효화, 안전한 쿠키 속성 설계.
