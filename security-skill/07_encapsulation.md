# 캡슐화 (P-01 ~ P-04)

## P-01 잘못된 세션에 의한 데이터 정보 노출 (CWE-488)
- **목적**: 한 세션/요청의 데이터가 다른 사용자에게 노출되지 않게.
- **공격 시나리오**: 요청 간 공유되는 가변 전역/싱글톤에 사용자 데이터 저장 → 교차 노출.
- **탐지 위치**: 모듈 전역 변수에 요청별 데이터 저장, 공유 가변 상태, 캐시 키에 사용자 식별자 누락.
- **PASS**: 요청 스코프 격리, 사용자별 캐시 키, 무상태 핸들러.
- **VULNERABLE**: 전역/싱글톤에 요청별 민감 데이터 저장.
- **수정**: 요청 컨텍스트 격리, 캐시 키에 사용자 범위 포함.

## P-02 제거되지 않고 남은 디버그 코드 (CWE-489)
- **목적**: 운영에 디버그/백도어 코드가 남지 않게.
- **공격 시나리오**: 디버그 엔드포인트/테스트 계정/`console.log(민감정보)` 잔존.
- **탐지 위치**: `/debug` 라우트, `console.log`/`print` 민감정보, 테스트 백도어, 디버그 플래그.
- **PASS**: 디버그 코드/엔드포인트 제거, 로그 레벨 분리.
- **VULNERABLE**: 운영 코드에 디버그/백도어 잔존.
- **수정**: 디버그 코드 제거, 환경별 빌드, 린트 규칙.

## P-03 Public 메소드로부터 반환된 Private 배열 (CWE-495)
- **목적**: 내부 가변 배열/객체 참조를 외부에 그대로 반환하지 않게.
- **공격 시나리오**: getter가 내부 배열 참조 반환 → 외부에서 내부 상태 변조.
- **탐지 위치**: getter가 내부 컬렉션/배열을 직접 반환.
- **PASS**: 방어적 복사 또는 불변 뷰 반환.
- **VULNERABLE**: 내부 가변 참조를 직접 노출.
- **수정**: 복사본/`readonly`/불변 자료구조 반환.

## P-04 Private 배열에 Public 데이터 할당 (CWE-496)
- **목적**: 외부에서 받은 가변 참조를 내부 private 필드로 그대로 보관하지 않게.
- **공격 시나리오**: setter가 외부 배열 참조를 내부에 저장 → 외부 변경이 내부에 반영.
- **탐지 위치**: setter/생성자가 매개변수 배열을 그대로 내부 필드에 할당.
- **PASS**: 입력 복사 후 저장.
- **VULNERABLE**: 외부 참조를 내부 상태로 직접 보관.
- **수정**: 방어적 복사 후 할당.
